更新最新版v3.3.1

版本发布说明
本次更新重点提升性能，引入新的输出显示单位，并扩展了 -m 参数选项，以提高配置灵活性。
关键更新
性能优化：已实现常规性能改进。
支持以兆字节 (M) 显示输出：现在可以以兆字节显示输出，以便更清晰地读取数据。
扩展的 -m 参数：添加了额外的内存模式，以适应不同的硬件配置。

2. -m 参数详细规范

下表列出了可用的 -m 参数模式：
参数 资源分配 显存需求 说明
-m 0 仅 GPU 需要超过 40GB 显存
-m 1 仅 GPU 需要超过 30GB 显存 
-m 2 仅 GPU 需要超过 22.5GB 显存
-m 3 仅 GPU 需要超过 21.25GB 显存
-m 42 混合（GPU + CPU） 每个实例分配约 3GB 显存。应与 export RUN_TASKS=N 环境变量一起使用，通过设置并发实例数 N 来优化资源，确保 3 * N 不超过显存总量。默认值为 RUN_TASKS=3。此模式对 PCIe 带宽要求更高。

3. 从 3.2.X 版本迁移指南

使用 -m 0 的用户：无需更改。
使用默认参数（无 -m 标志）运行的用户：无需更改，将使用新的默认设置。
之前的 -m 1 用户：您可能需要根据可用的 GPU 显存调整参数。请评估并从新选项中选择合适的模式：-m 1、-m 2 或 -m 3。
之前的 -m 2 用户：您有两个推荐选项：
切换到使用 -m 42 模式（并可选择通过设置 RUN_TASKS 环境变量进行微调）。
完全省略 -m 参数以使用新的默认配置。
